# VPN 連線故障排除標準作業程序（SOP）

> 文件編號：SOP-NET-VPN-001
> 適用對象：IT 維運 / 系統整合團隊
> 文件狀態：內部技術文件（範例，所有網段、主機名稱皆為虛構）

## 一、目的（Purpose）

本 SOP 規範企業 VPN gateway 的故障排除流程，協助維運人員快速處理
分公司與遠端使用者連線 timeout、斷線、無法存取內部系統（如 NAS、internal API）
等問題，縮短網路類事件（network_issue）的平均修復時間。

## 二、適用範圍（Scope）

- 總部與各分公司之間的站對站 IPsec VPN tunnel。
- 遠端員工使用的 VPN client 連線。
- VPN 推送的 DNS server 設定與內網路由。
- 與 firewall ACL、憑證（cert）到期相關的連線問題。
- 不含 VPN 設備本身的硬體更換流程。

## 三、常見症狀（Symptoms）

- VPN tunnel 在尖峰時段反覆斷線，連線 timeout。
- 使用者連上 VPN 後內部域名無法解析，DNS 查詢 timeout。
- 特定分公司透過 VPN 無法連到 NAS 或特定服務。
- VPN 同時連線數逼近授權上限，新使用者連線被拒絕無法登入。
- VPN client 更新後無法建立連線，錯誤訊息不明確。
- VPN 伺服器憑證即將到期，可能造成連線中斷。

## 四、可能原因（Possible Causes）

1. **協商與逾時設定**：IKE / IPsec 重連參數或 DPD timeout 設定不當，導致連線不穩。
2. **DNS 推送錯誤**：VPN 配置推送了錯誤的內部 DNS server，造成內網解析失敗。
3. **路由缺漏**：缺少對應分公司網段的靜態路由，封包無法到達 NAS 等內部資源。
4. **容量上限**：同時連線數達授權上限，新連線被拒。
5. **firewall 阻擋**：firewall ACL 規則誤擋 VPN 流量或內部服務埠。
6. **憑證問題**：VPN 伺服器或 client 憑證過期 / 不相符。
7. **用戶端版本**：VPN client 版本不相容或設定檔毀損。

## 五、故障排除步驟（Troubleshooting Steps）

### 5.1 確認故障範圍
- 釐清是單一使用者、單一分公司還是全面性問題。
- 檢視 VPN gateway 連線數與 tunnel 狀態，確認是否接近授權上限。
- 查看 VPN 與 firewall log，定位連線被 reset 或 timeout 的階段。

### 5.2 連線無法建立 / 反覆斷線
- 檢查 IPsec SA 是否正常協商，調整 DPD timeout 與重連參數。
- 確認兩端設定（加密演算法、預共享資訊）一致。
- 若為尖峰斷線，評估頻寬與連線數是否需擴充授權。

### 5.3 連上後無法存取內部資源
- 測試內部域名解析：確認 VPN 推送的 DNS server 正確。
- 檢查路由表是否包含目標網段（例如連到 NAS 的網段）。
- 與 firewall SOP 協同，確認 ACL 已放行 VPN 來源到目標服務的埠。

### 5.4 用戶端與憑證問題
- 確認 VPN client 版本，必要時重新安裝或回退至穩定版。
- 檢查伺服器與用戶端憑證有效期，於到期前完成更新（renew）。
- 重新匯入設定檔並清除毀損的本機快取。

## 六、升級判準（Escalation Criteria）

符合以下任一條件，應立即升級：

- 全公司或多個分公司 VPN 全面中斷且 30 分鐘內無法恢復。
- 連線數已達授權上限且短期內無法擴充，影響業務運作。
- 憑證已過期造成大規模連線中斷。
- 發現異常來源大量嘗試 VPN 登入，疑似攻擊（轉 security SOP 處理）。

## 七、預防檢查清單（Prevention Checklist）

- [ ] 建立憑證到期提醒，於到期前 30 天安排更新。
- [ ] 監控 VPN 同時連線數，接近授權上限時提前擴充。
- [ ] 定期驗證 VPN 推送的 DNS 與路由設定正確。
- [ ] 變更 firewall 規則前評估對 VPN 流量的影響。
- [ ] 維護一份相容且通過驗證的 VPN client 版本清單。
- [ ] 每季演練分公司斷線情境，確認備援連線可用。
